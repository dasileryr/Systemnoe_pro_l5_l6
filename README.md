# Задание 4: Приложение для запуска других приложений

## Описание

Приложение предоставляет пользователю возможность запуска других приложений:
- Блокнот (TextEdit на macOS)
- Калькулятор
- Paint (Preview на macOS)
- Своё собственное приложение

## Версии

### Версия 1: C++ (app_launcher.cpp)

Консольное приложение на C++ с интерактивным меню.

#### Компиляция

```bash
# Использование Makefile
make

# Или вручную
g++ -std=c++17 -Wall -Wextra -o app_launcher app_launcher.cpp
```

#### Запуск

```bash
./app_launcher
```

#### Очистка

```bash
make clean
```

### Версия 2: C# (app_launcher.cs)

Консольное приложение на C# с интерактивным меню.

#### Требования

- .NET SDK 6.0 или выше

#### Компиляция и запуск

```bash
# Компиляция и запуск одной командой
dotnet run

# Или компиляция отдельно
dotnet build

# Запуск скомпилированного приложения
dotnet run
```

#### Создание исполняемого файла

```bash
# Для текущей платформы
dotnet publish -c Release

# Для macOS
dotnet publish -c Release -r osx-x64 --self-contained false
```

## Использование

Обе версии имеют одинаковый функционал:

1. **Запуск стандартных приложений**: Выберите опцию из меню (1-3)
   - Опция 1: Блокнот (TextEdit)
   - Опция 2: Калькулятор
   - Опция 3: Paint (Preview)

2. **Запуск собственного приложения**: Выберите опцию 4 и введите путь к приложению

3. **Выход**: Выберите опцию 0

## Особенности реализации

### C++ версия
- Использует `system()` для запуска команд
- Поддержка macOS, Windows и Linux
- Консольный интерфейс с меню
- Обработка ошибок ввода

### C# версия
- Использует `Process.Start()` из `System.Diagnostics`
- Кроссплатформенность через `RuntimeInformation`
- Поддержка UTF-8 для кириллицы
- Обработка исключений

## Примечания

- На macOS приложения запускаются через команду `open`
- Для запуска собственных приложений можно указать полный путь к `.app` файлу или исполняемому файлу
- Обе версии работают в консольном режиме для максимальной совместимости
